# Print Labels

ThreadCloud prints a **hang tag** for every item you receive: barcode, size, colour, brand, and price. You print from the receiving screen. There are three ways to print, ranked best to fallback.

<div class="callout callout-info">
<div class="callout-title">Last verified against the app: 2026-06-29</div>
The hang tag's layout is set by the label designer in <strong>Settings</strong>, so the exact fields on your tag may differ from the example below.
</div>

## Option 1 — Dymo hang tag (preferred)

The default. A <strong>Dymo 30336</strong> label printer, <strong>54×25mm</strong> hang tags.

<ol class="steps">
  <li>Finish receiving (see <a href="/workflows/receiving/">Receiving</a>). On the success screen, click <strong>Print labels</strong>.</li>
  <li>A new tab opens with the hang tags — one per received unit (receive 5 of size L, you get 5 tags for that size).</li>
  <li>Press <kbd>Ctrl</kbd>+<kbd>P</kbd>.</li>
  <li>Select the Dymo printer; confirm the size is the Dymo 30336 (54×25mm).</li>
  <li>Print.</li>
</ol>

### What's on the hang tag

A typical tag shows:

- The **barcode** (the per-size variant barcode) and the SKU
- **Size** (large, easy to scan during a fitting)
- **Colour**
- **Brand** and the style code
- **Price**
- Optional badges (e.g. stock type, "Dry Clean", "Made in Italy") if your tag design includes them

<div class="callout callout-tip">
<div class="callout-title">Change what's on the tag</div>
The hang tag is configurable. Open <strong>Settings</strong> and find the label designer to choose which fields print, adjust the layout, and set the printer. The Owner usually sets this up once.
</div>

## Option 2 — Avery 5160 sheet labels (printer-agnostic fallback)

If the Dymo dies, runs out, jams, or isn't available.

<ol class="steps">
  <li>Buy Avery 5160 sheet labels (30-up, 1×2⅝ inches) from any office supply store.</li>
  <li>Load them into any regular laser/inkjet printer.</li>
  <li>Click <strong>Print labels</strong> in ThreadCloud as usual.</li>
  <li>In the print dialog: select your normal printer, set paper to Letter, scale to 100%.</li>
  <li>Print.</li>
</ol>

<div class="callout callout-warn">
<div class="callout-title">Heads up</div>
The tag is sized for the Dymo. On Avery sheets you'll get the same tag but the layout shifts (one tag per slot, with whitespace). It's a working fallback, not a perfect match. Test one sheet before printing a stack.
</div>

## Option 3 — Shopify Admin barcode print (last resort)

When neither the Dymo nor a backup is available. This works because ThreadCloud writes the SKU into the Shopify variant `barcode` field on every push.

<ol class="steps">
  <li>Open Shopify Admin → Products.</li>
  <li>In the search bar, type <code>tag:tc-order-<strong>247</strong></code> — replace 247 with your order's ID (find it in the ThreadCloud order URL).</li>
  <li>Select the rows that need labels.</li>
  <li>Click <strong>More actions</strong> → <strong>Print barcode labels</strong>.</li>
  <li>Choose a label size (Avery 5160 is safest), choose products and variants, print.</li>
</ol>

You lose brand and colour, but keep size, SKU, barcode, and price — enough to identify and ring up each item.

## Reprinting tags later

You don't have to print at receive time. To reprint:

- Open <strong>Logistics</strong> and use the <strong>Reprint Tags</strong> tile — it lands on the shipment so you can print its tags again.
- Or open the order's receiving log, find the shipment, and print from there.

This reprints the tags for that shipment. To reprint just a few, print the sheet and discard the rest, or use the Shopify Admin fallback to cherry-pick by variant.

## If it goes wrong

<div class="callout callout-warn">
<div class="callout-title">Print scale set to "Fit to page"</div>
This shrinks the barcode below scannable size. Set scale to 100% / Actual Size in the print dialog.
</div>

<div class="callout callout-warn">
<div class="callout-title">Tags won't print after a while</div>
The print link is time-limited. If it stops working, go back to the shipment and open <strong>Print labels</strong> again to get a fresh link.
</div>
